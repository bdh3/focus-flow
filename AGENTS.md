# Focus Flow AI Agent 지침서 (v1.8.0)

본 문서는 Focus Flow 프로젝트를 수행하는 AI 에이전트가 반드시 준수해야 할 기술적/정책적 가이드라인을 정의합니다. 모든 작업 전에 이 파일을 참조하여 프로젝트의 일관성을 유지해야 합니다.

## 0. 최우선 운영 원칙 (승인 절차)
- **명시적 승인 필수**: Git 커밋, 태그 생성, 푸시, 버전 상향 등 모든 배포 관련 작업은 반드시 사용자의 **명시적인 승인(예: "윤허한다", "진행해")**을 확인한 뒤에만 수행합니다. 제안 후 답변을 기다리지 않고 명령을 실행하는 것은 금지됩니다.

## 1. 핵심 개발 원칙
- **버전 관리 엄수**: 프로젝트의 현재 버전(v1.8.0)을 반드시 확인하고 임의로 버전을 올리지 마십시오. 
- **도구 사용 제한 (절대 금지)**:
    - 파일 수정은 반드시 `replace_file_content`, `multi_replace_file_content`, `write_file` 도구를 사용하십시오. 
- **일관성 우선**: 현대적인 Android 개발 관례를 따르되, 기존 코드의 스타일과 명명 규칙을 최우선으로 존중하십시오.

## 2. 네비게이션 및 UI 상태 관리
- **백스택 관리**: 하단 탭(Timer, Calendar, Settings) 간 이동 시 백스택이 무한히 쌓이지 않도록 `launchSingleTop = true` 및 `popUpTo(startDestination)` 옵션을 반드시 적용합니다. 뒤로가기 시 탭이 순차적으로 나오지 않고 바로 앱이 종료되거나 홈으로 가야 합니다.
- **앱 재진입 시 초기화**: 타이머가 동작 중이 아닐 때 앱을 완전히 나갔다 들어오면, 이전 세션 정보(선택된 작업 등)를 초기화하여 **"작업을 선택하세요"** 상태로 깨끗하게 시작해야 합니다.

## 3. 알람 및 알림 시스템 (README_ALARM.md v1.8.0 준수)
- **핵심 원칙**: 알람 트리거, 모드(전체 화면/팝업), 사운드 제한, 타임아웃(20초), 중지 조건 등 모든 세부 로직은 **`README_ALARM.md`**의 최신 명세를 절대적으로 따릅니다.
- **용어 개편**: '몰입' -> **'집중'**, '독립 세션' -> **'집중 세션'**으로 전면 교체하여 사용자 친화성을 향상합니다. 작업명이 없는 경우 **"집중 세션"** 명칭을 사용하며, 타이틀은 `작업명: 상태` 형식을 따릅니다.
- **우선순위**: 벨소리 사운드이거나 유저 설정이 '전체 화면'인 경우, 시스템 환경과 무관하게 전체 화면 알람을 보장합니다.
- **안정성**: 종료 알람(`isFinished`) 발생 시에도 `setOngoing(true)`를 유지하여 알람의 우선순위를 확보하고 시스템에 의해 사라지지 않도록 합니다.

## 4. UI/UX 디자인 가이드라인
- **알람 UI**: `AlarmActivity`의 중단 버튼을 **80dp** 높이로 확대하여 터치 편의성을 증대하고, 다크모드 배경색을 `0xFF0F172A`로 고정하여 야간 눈부심을 방지합니다.
- **설정 디자인**: 뽀모도로 프리셋(25/5, 50/10)을 카드/서피스 스타일의 직관적인 디자인으로 구성합니다.
- **캘린더 월간 뷰**: 날짜별 작업 숫자 뱃지는 원형 중앙에 정렬되어야 하며, 9.sp / Black 굵기로 가독성을 확보합니다.
- **완료된 작업**: 흐린 배경(SurfaceVariant 0.5f) + **취소선(LineThrough)** 처리.
